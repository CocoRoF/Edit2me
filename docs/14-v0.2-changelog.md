## 사용자에게 보이는 변화

### CJK PDF
- **PR #1**: Adobe-Korea1/Japan1/GB1/CNS1 CID→Unicode 매핑 infrastructure. ToUnicode 부재 PDF에서도 Latin region은 즉시 디코드. `npm run build:cmaps` 실행 시 Adobe 공식 CMap 데이터를 fetch해 한글/한자/가나 전 영역 디코드 가능.

### 편집 정확도
- **PR #2**: 텍스트 편집으로 길이가 변해도 같은 줄의 후속 텍스트가 어긋나지 않음 (`Td` 자동 보정).

### 메모리
- **PR #2**: PdfDocument의 객체 캐시가 LRU(4096) — 큰 PDF에서 OOM 방지.

### Undo/Redo
- **PR #3**: ⌘Z / ⌘⇧Z / ⌘Y 또는 toolbar 버튼. 서버에서 op replay 기반.

### 큰 PDF 스크롤
- **PR #4**: 보이지 않는 페이지의 텍스트 블록 DOM 미생성 (IntersectionObserver). 100+ 페이지에서도 메모리/레이아웃 비용이 viewport에 비례.

### UX 폴리시
- **PR #5**: 진단 사이드 패널 — 폰트별/페이지별 warning 집계 ("페이지 1–3, 7–8" 같은 range 압축). Toolbar 경고 배지. 키보드 단축키 모달 (?).
- **PR #6**: 모바일 햄버거 + 사이드바 drawer. Toolbar/사이드바 모두 좁은 화면 적응.
- **PR #7**: 한국어/영어 토글 (Toolbar 우측 globe 아이콘).

### 견고성
- **PR #8**: tokenizer / LRU / CMap parser / CID lookup / serializer 의 단위 테스트 29개. v0.1 의 whitespace-in-hex 버그를 회귀 테스트로 잠금.

## 미완 / 보류 (v0.3 후보)

[`docs/13-quality-review.md` §2](./13-quality-review.md) 의 우선순위 목록 참고. 주요 항목:

- A4 잔여 — `npm run build:cmaps` 결과를 컨테이너 빌드 단계에서 자동 실행 (현재는 수동)
- A7 — content stream parser 최적화 (큰 콘텐츠 stream의 GC 압력)
- A8 — vertical writing mode (한자 vertical PDF)
- C5 — surgical update (op 적용 후 client 가 doc 메타 전체 재요청 안 하도록)
- D3 — 사용자 TTF subset embedding (한글 텍스트 *추가*)
- D4 — 회전된 페이지에서 텍스트 추가 좌표 보정
- E1 — `as` 단언 줄이기 + zod 등 validation 도입 검토
- E2 — 클라이언트 fetch 결과 SWR/RSC 캐시 레이어
- B7 잔여 — 활성 페이지 자동 스크롤 (현재는 사용자 스크롤만)
- 자체 raster renderer (Phase 7) — 시각 충실도가 사용자 가치에 결정적이라 판단되면 시작
