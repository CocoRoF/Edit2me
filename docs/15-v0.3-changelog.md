## 사용자에게 보이는 변화

### Vertical 한자/한국어 PDF
- **PR #10**: `/WMode 1` (또는 `-V` suffix CMap) 폰트가 들어간 PDF에서 텍스트가 *세로 방향* 으로 정확한 위치에 잡힘.

### 한국어 텍스트 *추가* 가능 — D3 완료
- **PR #16 + #17**: AddTextDialog 에 "TTF 추가" 버튼. 사용자가 한글 폰트(예: Noto Sans KR, Pretendard)를 업로드하면 텍스트 추가 시 한글이 그대로 PDF에 임베드. Type0 + CIDFontType2 + Identity-H + ToUnicode CMap → 다른 뷰어에서도 검색·복사 가능.

### 진단 시스템
- **PR #11**: 정상 PDF에도 정보 진단(`pdf-version`, `incremental-history`)이 진단 패널에 표시. 손상 케이스(`header-offset`)는 경고로.

### 회전 페이지 편집
- **PR #13**: `/Rotate 90/180/270` 페이지에서 더블클릭으로 텍스트를 추가하면 사용자가 *본* 위치에 정확히 떨어짐.

### 성능 / 코드 품질
- **PR #12**: ops 응답이 페이지 스냅샷을 포함 → 매 op 후 GET /documents 호출 제거. 응답 1번에 모든 정보.
- **PR #14**: API 경계의 op-validate 로 잘못된 클라이언트 요청을 일찍 거절. 클라이언트 fetch 캐시 추가 (60s TTL, 명시적 invalidation).

### 운영
- **PR #15**: 호스트 Dockerfile 이 `npm run build:cmaps` 를 빌드 단계에 자동 실행 (ARG 로 끄기 가능). build-cmaps.mjs 의 OUT_DIR 자동 감지로 컨테이너에서도 정확한 경로에 출력.

## 자체 raster renderer — 의도적 보류

[`docs/05-renderer.md`](./05-renderer.md) Phase 2~3 의 자체 raster renderer는 본 사이클에서 시작하지 않았다. 자세한 결정 근거는 [`docs/adr/0004-raster-renderer-deferred.md`](./adr/0004-raster-renderer-deferred.md) 참고.

요약: 자체 raster 는 글리프 outline 라스터화(스캔라인) + 그래픽 상태 + 색공간 + 이미지 디코드 까지 구현해야 하는 multi-week 작업이고, **현재 사용자 가치 대비 비용이 크다**. 사용자가 "시각 미리보기" 가 결정적이 되면 그때 별도 사이클로 시작.
