# 16. v0.4 Changelog — Vector Renderer

> v0.3 ([commit `270c128`](https://github.com/CocoRoF/Edit2me/commit/270c128)) 이후의 PR 5개로 SVG 기반 벡터 렌더러를 도입. 사용자가 PDF 의 *진짜 모양* (텍스트뿐 아니라 표·박스·도장·이미지·임베디드 폰트의 글자 outline) 을 보게 됨.

## 동기

v0.3 까지의 PageView 는 *텍스트 오버레이 only* — 도형/이미지가 안 보이는 흰 박스. 사용자 보고: "PDF의 모든 SVG 라인을 전부 렌더링 해야 하는데 장난해?" v0.3 의 ADR-0004 (raster renderer 보류) 결정을 *재논의* 해 SVG 방향으로 전환했다.

## 사용자에게 보이는 변화

### 가장 큰 변화 — PDF 의 진짜 모양
- 학위증명서/성적증명서/계약서 같은 표·라인 기반 PDF: 표 박스, 구분선, 도장 이미지가 모두 정확한 위치에 표시됨.
- 한국어 PDF: 임베디드 폰트의 글자 outline 이 SVG path 로 변환되어 OS 폰트로 근사하지 않은 *원본과 동일한 모양*.
- 이미지: JPEG (DCTDecode) 는 그대로, raw 픽셀은 자체 PNG 인코더로 표시.

### Form XObject 재귀
- 페이지가 공통 머리말/꼬리말/도장을 Form XObject 로 재사용하는 경우, 그 내부 ops 까지 재귀적으로 렌더.

### Color
- DeviceRGB/Gray/CMYK 외에 **Indexed ColorSpace** (palette-based) 본격 지원. 한국 정부 양식이 자주 사용.

## 엔진 아키텍처

```
pdf/render/
├── svg-renderer.ts   ← 메인. PdfDocument + page → SVG 문자열
├── png-encoder.ts    ← 8-bit gray/RGB PNG 인코더 (raw image용)
└── (future)
    └── (raster fallback if needed)

pdf/fonts/
├── ttf-glyph.ts      ← TTF glyf 테이블 → SVG path d (simple + composite)
└── ttf-parser.ts     ← head/maxp/hhea/hmtx/cmap (기존)

pdf/graphics/content-stream.ts ← 모든 PDF op 인식 (path/paint/color/clip/...)
```

처리 흐름:
1. `parseContent(byte stream)` → `ContentOp[]`
2. `runOpsScope(ops, depth)` 가 graphics state 머신을 돌리며 SVG element 발행
3. Op 별:
   - Path 구성 (m/l/c/v/y/re/h) → 내부 PathBuilder
   - 페인트 (S/f/B/...) → `<path d="...">`
   - Image XObject → `<image href="data:...">` (JPEG 패스스루 또는 PNG 인코드)
   - Form XObject → 자체 ops 재귀 처리 (자식 resources 우선)
   - Text → 글리프별 `<path>` (outline 가능 시) 또는 `<text>` (OS 폰트 fallback)

좌표:
- 외곽 group 의 `matrix(1,0,0,-1,0,H)` 로 PDF y-up → SVG y-down 한 번만 flip.
- 텍스트는 outer flip 안에서 거꾸로 안 그려지게 자체 y-flip (`matrix(a,b,-c,-d,e,f)`).

## 의도적 한계 (v0.5 후보)

| 영역 | 보류 이유 |
|---|---|
| OpenType/CFF (OTTO scaler) 폰트 outline | CFF Type2 charstring 파서 필요 — 별도 모듈. 한국어 PDF 의 99% 가 TrueType 라 실용 영향 적음. |
| Type 3 폰트 (콘텐츠 스트림 글리프) | 희귀. fallback 으로 OS 폰트. |
| Pattern fill (격자 등) | Pattern dict 파싱 + 타일 SVG 생성 — 큰 작업. fallback 균일색. |
| Shading (gradient) | linearGradient/radialGradient 변환 가능하나 dict 파서 필요. fallback 균일색. |
| Soft mask / Transparency group | SVG `<mask>` 또는 filter 활용 가능 — 별도 PR. |
| ICCBased 색공간 정확도 | ICC profile 파서 필요. 현재 Device 로 fallback. |
| 스캔 이미지의 CCITTFax / JBIG2 | 디코더 별도 구현 (비트 단위). 회색 박스 fallback. |
| Annotations (주석/하이라이트) overlay | v0.5+. |

## ADR 상태 변경

[`docs/adr/0004-raster-renderer-deferred.md`](./adr/0004-raster-renderer-deferred.md) 의 결정은 v0.4 에서 *부분 재론*. raster (canvas) renderer 는 여전히 보류이지만 SVG renderer 로 같은 시각 가치를 달성. raster 는 인쇄 용도/픽셀 정확도가 필요할 때 고려.
