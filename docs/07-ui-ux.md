# 07. UI/UX Design

## 화면 흐름

```
[/edit2me]                          ← 랜딩 (업로드)
   │
   ├─ 단일 파일 → /edit2me/e/{docId}    ← 에디터
   │
   └─ 다중 파일 → /edit2me/m            ← 병합 모드
                     │
                     └─ 완료 → /edit2me/e/{docId}   (병합 결과를 다시 편집 가능)
```

라우트는 모두 `basePath: /edit2me` 아래 동작.

## 1. 랜딩 (`/`)

```
┌────────────────────────────────────────────────────────────┐
│ Edit2me                                            [한 EN]  │
├────────────────────────────────────────────────────────────┤
│                                                            │
│             ┌────────────────────────────────┐             │
│             │                                │             │
│             │   PDF를 끌어다 놓거나 클릭     │             │
│             │                                │             │
│             │   [파일 선택]   [예제 열기]    │             │
│             │                                │             │
│             └────────────────────────────────┘             │
│                                                            │
│  ┌─ 무엇을 할 수 있나요 ──────────────────────────────┐   │
│  │  ✎  텍스트 편집      ⤵ 페이지 재배치/삭제          │   │
│  │  ✚  텍스트 추가      ⊞ 여러 PDF 병합              │   │
│  └───────────────────────────────────────────────────┘   │
│                                                            │
│  단일 파일 → 편집 모드 / 여러 파일 → 병합 모드             │
│                                                            │
│  업로드한 파일은 24시간 후 자동 삭제됩니다.                │
│  외부 PDF 라이브러리를 사용하지 않는 자체 엔진으로 동작.   │
└────────────────────────────────────────────────────────────┘
```

**핵심 UX**:
- 드래그 드롭 영역이 화면 중앙에 큼지막하게.
- 파일 1개 → 자동으로 에디터 진입.
- 파일 N개 → 병합 모드로.
- 진행중 progress 바 + 파싱 단계 표시 ("스캔 중... 폰트 분석... 텍스트 추출...").

## 2. 에디터 (`/e/{docId}`)

```
┌────────────────────────────────────────────────────────────────────┐
│ Edit2me  파일명.pdf                              [↶][↷][다운로드 ▾]│ ← Toolbar
├──────┬─────────────────────────────────────────────────────────────┤
│      │                                                             │
│ [페] │                                                             │
│ ┌──┐ │                                                             │
│ │ 1│ │                                                             │
│ └──┘ │                  ┌─────────────────────────┐               │
│ ┌──┐ │                  │                         │               │
│ │ 2│ │                  │                         │               │
│ └──┘ │                  │     페이지 컨텐츠        │               │
│ ┌──┐ │                  │   (텍스트 오버레이)      │               │
│ │ 3│ │                  │                         │               │
│ └──┘ │                  │                         │               │
│ ┌──┐ │                  └─────────────────────────┘               │
│ │ 4│ │                                                             │
│ └──┘ │                                                             │
│      │                                                             │
│ [+]  │                                                             │
│      │                                                             │
├──────┴─────────────────────────────────────────────────────────────┤
│  [✎] [✚텍스트] [⟲회전]  ......  100% [-/+]   준비됨            ▢▢ │ ← Statusbar
└────────────────────────────────────────────────────────────────────┘
```

### 좌측 사이드바: 페이지 보드

- 가로 110px, 페이지 썸네일 카드.
- 카드 표시:
  - 1차: 페이지 번호 + 비율 박스 + 첫 5개 텍스트 블록을 작은 글씨로.
  - Phase 2: 실제 raster 미리보기.
- 다중 선택: Cmd/Ctrl + 클릭, Shift + 클릭.
- 우클릭: "삭제", "복제", "회전 →", "회전 ←".
- 드래그: 카드를 잡아 다른 위치로 떨어뜨림. 빨간 가이드 라인.
- 하단 [+]: 페이지 가져오기 (다른 PDF에서).

### 메인 영역: 페이지 캔버스

- 활성 페이지의 텍스트 오버레이.
- 줌: Ctrl+휠, Cmd+0(맞춤), Cmd++/Cmd+-.
- 가로 가운데 정렬. 위/아래 빈 공간으로 스크롤 시 다음/이전 페이지로 자동 이동.
- 다중 페이지를 동시에 표시할 수 있는 "한 페이지 / 두 페이지 / 흐름 보기" 토글 — 데스크톱은 흐름 보기 default.

### 텍스트 편집 모드

- 텍스트 블록 hover → 점선 보더 + 커서 변화.
- 더블클릭 → contenteditable. 글자 입력 가능.
- 편집 중에는 *해당 블록 외 모든 영역에 light overlay* — 포커스 강조.
- 이탈 = 자동 commit. ESC = 취소.

### 텍스트 추가 모드

- 툴바의 "✚ 텍스트" 클릭 → 마우스 커서가 십자.
- 페이지 빈 곳 클릭 → 빈 입력 박스 출현, 즉시 포커스.
- 입력 박스 옆에 mini-toolbar: 폰트 / 크기 / 색.
- ESC = 취소, Enter = commit (Shift+Enter는 줄바꿈).

### Top Toolbar

- 왼쪽: 파일명 (클릭하면 새 파일 열기).
- 오른쪽: Undo / Redo / 다운로드 ▾
  - 다운로드 메뉴: "원본 변경분만 추가 (안전, 큼)" / "최적화 (작음)" — 기본은 전자.

### Status Bar

- 현재 페이지 번호 / 전체.
- 줌.
- "변경됨" / "저장됨" 상태.
- 진단(`diagnostics`) 카운트 — 클릭하면 패널 슬라이드인.

## 3. 병합 모드 (`/m`)

```
┌────────────────────────────────────────────────────────────────────┐
│ Edit2me   병합                              [↶][↷][병합 완료 ▷]   │
├────────────────────┬───────────────────────────────────────────────┤
│ [+] 파일 추가      │                                               │
│                    │           결과 시퀀스                          │
│ ▼ doc1.pdf         │   ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐                  │
│   [1][2][3][4]     │   │ 1│ │ 2│ │ 3│ │ 4│ │ 5│                  │
│                    │   └──┘ └──┘ └──┘ └──┘ └──┘                  │
│ ▼ doc2.pdf         │                                               │
│   [1][2][3]        │   드래그로 추가                               │
│                    │                                               │
│ ▶ doc3.pdf         │                                               │
└────────────────────┴───────────────────────────────────────────────┘
```

- 좌측에 업로드된 파일 별 페이지 카드 그리드.
- 우측에 결과 시퀀스. 빈 상태에는 "← 왼쪽에서 페이지를 끌어다 놓으세요".
- 결과 시퀀스 내에서도 카드를 드래그로 재정렬, Backspace로 제거.
- "병합 완료" → 서버에서 `merge` op 실행 → 다운로드 옵션 표시.

## 4. 단축키

| 작업 | Mac | Win/Linux |
|---|---|---|
| Undo | `⌘Z` | `Ctrl+Z` |
| Redo | `⌘⇧Z` | `Ctrl+Shift+Z` |
| 줌 맞춤 | `⌘0` | `Ctrl+0` |
| 100% | `⌘1` | `Ctrl+1` |
| 다음/이전 페이지 | `J / K` (또는 PageDown/PageUp) | 동일 |
| 페이지 삭제 | `⌫` (사이드바 포커스 시) | `Delete` |
| 텍스트 추가 모드 | `T` | `T` |
| 다운로드 | `⌘S` | `Ctrl+S` |
| 검색 (v2) | `⌘F` | `Ctrl+F` |

## 5. 시각 디자인

- hr_blog2.0의 Tailwind 4 토큰을 *재사용*해서 일관성 확보. 색 팔레트 / 간격 / radius / 타이포그래피.
- 다크 모드 1차 지원: hr_blog2.0의 `prefers-color-scheme` 따름.
- Lucide 아이콘 (hr_blog2.0이 이미 사용 중이라 무료).

## 6. 접근성

- 모든 인터랙션을 키보드로 수행 가능. 사이드바 포커스 → 화살표/Delete 등.
- ARIA: 페이지 카드는 `role="button"`, 다중 선택은 `aria-multiselectable`.
- 텍스트 편집 시 contenteditable에 적절한 ARIA role.
- 색에만 의존한 정보 전달 금지 (예: "수정됨" 배지는 색 + 텍스트 둘 다).
- 폰트 크기 ≥ 14px UI 기본.

## 7. 에러 / 빈 / 로딩 상태

| 상태 | 표시 |
|---|---|
| 업로드 중 | 큰 progress + 단계 ("xref 분석..." 등) |
| 파싱 실패 | 친화적 에러 + "복구 시도" 버튼 + "다른 파일 열기" |
| 암호화 PDF | "비밀번호 보호된 PDF는 v1에서 지원하지 않습니다" + 도움 링크 |
| 폰트 미지원 (편집) | 텍스트 블록을 *읽기 전용*으로 표시 + 작은 자물쇠 아이콘 |
| 변경 후 미저장 페이지 이탈 | beforeunload 경고 + 자동 5초 저장 안내 |
| 다운로드 직전 검증 실패 | 모달 "저장 중 검증 실패. 자세히 보기 / 다시 시도" |

## 8. 모바일

- v1: 반응형 레이아웃이지만 "에디터는 데스크톱 권장" 안내. 모바일에서는 페이지 *재배치/삭제* + *다운로드*만 활성. 텍스트 편집/추가는 데스크톱.
- 하단 탭바로 사이드바/메인/툴바 전환.
